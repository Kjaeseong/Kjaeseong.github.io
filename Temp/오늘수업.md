### Process
- 현재 실행되고 있는 노드 프로세스에 대한 정보를 담고 있음
- 실행 빈도는 높지 않지만 일반저그올 운영체제나 실행 환경별로 다른 동작을 하고 싶을 때 사용
- 아래 명시되는 것들은 중요

#### process.env
- 시스템의 환경 변수
- 노드에 직접 영향을 미치기도 한다.
- UV_THREADPOOL_SIZE, NODE_OPTIONS가 대표적
  - NODE_OPTIONS : 노드를 실행할 때 옵션들을 입력받는 환경 변수
- 서비스의 중요한 키를 저장하는 공간으로도 사용
  - 서버, 데이터베이스의 비밀번호, API 키를 코드에 직접 입력하는 것은 위험
  - 중요한 속성은 다음과 같이 process.env의 속성으로 대체
  
```javascript
const secretId = process.env.SECRET_ID;
const secretCode = process.env.SECRET_CODE;
```

#### process.nextTick(콜백)
- 이벤트 루프가 다른 콜백 함수들보다 nextTick의 콜백 함수를 우선적으로 처리하도록 한다.
- process.nextTick과 Promise를 마이크로태스크(microtask)라고 구분지어 부른다.
- 마이크로태스크의 재귀 호출
  - 다른 콜백 함수보다 마이크로태스크를 우선하여 처리
  - 콜백 함수들이 실행되지 않을 수도 있다.

```javascript
setImmediate(() => {
    console.log('immediate');
});
process.nextTick(() => {
    console.log('nextTick');
});
setTimeout(() => {
    console.log('timeout');
}, 0);
Promise.resolve().then(() => console.log('promise'));
```


#### process.exit(코드)
- 실행 중인 노드 프로세스를 종료
- 서버 환경에서 사용시 서버가 멈춘다.
  - 특수한 경우를 제외하고 서버에서 사용하지 않는다
- 서버 외의 독립적인 프로그램에서 수동으로 노드를 멈추기 위해 사용
- 

```javascript
let i = 1;
setInterval(() => {
    if (i === 5) {
        console.log('종료');
        process.exit();
    }
    console.log(i);
    i += 1;
}, 1000);
```

### 노드 내장 모듈 사용하기

#### os
- 운영체제의 정보가 담겨 있어 정보를 가져올 수 있다
  - 웹 브라우저의 js는 운영체제의 정보를 가져올 수 없다
- os.cpus().length 사용시 코어의 개수가 숫자로 반환
  - 노드에서 싱글 스레드 프로그래밍을 하면 코어가 몇개든 상관없이 코어를 하나밖에 사용하지 않는다.
  - cluster 모듈 사용시 코어 개수에 맞춰서 프로세스를 늘릴 수 있음
- 코드 넣을것(113p)

#### path
- 폴더와 파일의 경로를 쉽게 조작하도록 도와주는 모듈
  - 필요한 이유 : 운영체제별로 경로 구분자가 다르기 때문
  - Windows : \
  - POSIX : /
- 코드넣자 116p
- 추가설명도

#### url
- 인터넷 주소를 쉽게 조작하도록 도와주는 모듈
  - WHATWG(웹 표준을 정하는 단체 이름) 방ㅅ식
  - 예전부터 노드에서 사용하던 방식
