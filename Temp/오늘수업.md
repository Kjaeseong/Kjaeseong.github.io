### Process
- 현재 실행되고 있는 노드 프로세스에 대한 정보를 담고 있음
- 실행 빈도는 높지 않지만 일반저그올 운영체제나 실행 환경별로 다른 동작을 하고 싶을 때 사용
- 아래 명시되는 것들은 중요

#### process.env
- 시스템의 환경 변수
- 노드에 직접 영향을 미치기도 한다.
- UV_THREADPOOL_SIZE, NODE_OPTIONS가 대표적
  - NODE_OPTIONS : 노드를 실행할 때 옵션들을 입력받는 환경 변수
- 서비스의 중요한 키를 저장하는 공간으로도 사용
  - 서버, 데이터베이스의 비밀번호, API 키를 코드에 직접 입력하는 것은 위험
  - 중요한 속성은 다음과 같이 process.env의 속성으로 대체
  
```javascript
const secretId = process.env.SECRET_ID;
const secretCode = process.env.SECRET_CODE;
```

#### process.nextTick(콜백)
- 이벤트 루프가 다른 콜백 함수들보다 nextTick의 콜백 함수를 우선적으로 처리하도록 한다.
- process.nextTick과 Promise를 마이크로태스크(microtask)라고 구분지어 부른다.
- 마이크로태스크의 재귀 호출
  - 다른 콜백 함수보다 마이크로태스크를 우선하여 처리
  - 콜백 함수들이 실행되지 않을 수도 있다.

```javascript
setImmediate(() => {
    console.log('immediate');
});
process.nextTick(() => {
    console.log('nextTick');
});
setTimeout(() => {
    console.log('timeout');
}, 0);
Promise.resolve().then(() => console.log('promise'));
```


#### process.exit(코드)
- 실행 중인 노드 프로세스를 종료
- 서버 환경에서 사용시 서버가 멈춘다.
  - 특수한 경우를 제외하고 서버에서 사용하지 않는다
- 서버 외의 독립적인 프로그램에서 수동으로 노드를 멈추기 위해 사용
- 

```javascript
let i = 1;
setInterval(() => {
    if (i === 5) {
        console.log('종료');
        process.exit();
    }
    console.log(i);
    i += 1;
}, 1000);
```

#### 단방향 암호화
- 복호화할 수 없는 암호화 방식
  - 복호화 : 암호화된 문자열을 원래 문자열로 되돌려놓는 것
- 주로 해시 기법을 사용
  - 해시 함수라고 부르기도 한다
- 유저의 비밀번호 자체는 저장하지 않고, 암호화된 문자열로만 비교, 데이터베이스의 비밀번호와 비교
- createHash(알고리즘) : 
  - 사용할 해시 알고리즘.
  - md5, sha1, sha256, sha512등 가능
    - md5, sha1은 취약점이 발견
  - 추후 sha512마저 취약해지면 더 강화된 알고리즘으로 바꿔야 한다.
- update(문자열) : 변환할 문자열
- digest(인코딩) :
  - 인코딩할 알고리즘. 변환된 문자열을 반환
  - base64, hex, latin1 주로 사용
  - base64가 결과 문자열이 가장 짧아 애용

#### 양반향 암호화
- 암호화된 문자열 복호화 가능
- '키' 사용.
- 복호화 시 암호화할 때 사용한 키와 같은 키를 사용해야 한다.
- 완벽한 이해를 위해 암호학을 추가로 공부해야 한다.

```javascript
const crypto = require('crypto');

const algorithm = 'aes-256-cbc';
const key = 'abcdefghijklmnopqrstuvwxyz123456';
const iv = '1234567890123456';

// 암호화
const cipher = crypto.createCipheriv(algorithm, key, iv);
let result = cipher.update('암호화할 문장', 'utf8', 'base64');
result += cipher.final('base64');
console.log('암호화', result);

// 복호화
const decipher = crypto.createDecipheriv(algorithm, key, iv);
let result2 = decipher.update(result, 'base64', 'utf8');
console.log('복호화', result2);
```

- localhost : 현재 컴퓨터의 내부 주소. 자신의 컴퓨터에서만 접근할 수 있다. 서버 개발시 테스트용으로 많이 사용

```javascript
// server1-1.js

const http = require('http');

const server = http.createServer((req, res) => {
    res.writeHead(200, { 'content-Type': 'text/html; charset=utf-8' });
    res.write('<h1>hello Node!</h1>');
    res.end('<p>Hello Server!</p>');
});
server.listen(8080);

server.on('listening', () => {
    console.log('8080 포트에서 서버 대기 중');
});
server.on('error', (error) => {
    console.error(error);
});
```
