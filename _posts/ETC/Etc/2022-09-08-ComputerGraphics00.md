# 컴퓨터 그래픽스 기초

## 1. 기초지식
- 기초지식은 [삼성 디스플레이](https://news.samsungdisplay.com/305)에서 정리를 잘 해놨다.

### 1.1. 픽셀과 셰이더

#### 1.1.1. 픽셀
- 픽셀 : 디지털 이미지를 구성하는 최소 단위( = 화소)
- 가산혼합 : R, G, B로 서브픽셀 구성, 혼합해 원하는 색을 만든다
  - 불투명도를 나타내기 위한 알파채널 포함
  - 각 채널은 0 ~ 1 사이의 실수로 표현
  - *서브픽셀 : 1픽셀 이내의 R, G, B가 각 서브픽셀.
- 각 채널을 통과하는 빛의 양으로 계조(Grayscale)표현


#### 1.1.2. 쉐이더(Shader)
- 최종적으로 화면에 출력할 픽셀의 색을 정하는 함수
- 유니티에서 쉐이더를 사용하기 위한 두 가지 방법
  - 쉐이더 그래프 : 시각적으로 다루는 툴
  - 쉐이더랩 : 코드 작성

## 2. 렌더링 파이프라인
- ( = Graphic Pipeline)
- 3D물체를 화면에 출력하기 위한 일련의 과정
- 그래픽스 API에 따라 다를 수 있다.
  - [OpenGL](https://www.khronos.org/opengl/wiki/Rendering_Pipeline_Overview)
  - [Direct3D 11](https://docs.microsoft.com/ko-kr/windows/win32/direct3d11/overviews-direct3d-11-graphics-pipeline)


### 2.1. 버텍스 로드
- 3D 물체가 어떻게 생겼는지에 대한 데이터
  - 물체를 구성하는 점
- 렌더링을 위한 여러 속성
  - 인덱스 번호(Index Number)
  - 위치(Position)
  - 노멀(Normal)
  - 색(Color)
  - UV
- 속성 정보를 받아 삼각형을 그려 물체의 기본적인 형태를 만든다.
  - 유일하게 삼각형만이 오직 평면 표현 가능
  - 모든 다각형은 삼각형으로 표현 가능

### 2.2. 버텍스 쉐이더
- 버텍스와 관련된 각종 연산
- 버텍스 라이트라는 매우 가볍고 부정확한 라이트가 연산될 수 있다. 여기에는 고라우드 쉐이딩(Gouraud Shading)이라는 것이 있다.

| 과정    | 설명                                                                             |
|-------|--------------------------------------------------------------------------------|
| 월드 변환 | 각 물체는 로컬좌표를 갖는다<br/>다른 물체와 상호작용을 위해 월드공간으로 변환(World Transform)이라 한다.           |
| 뷰 변환  | 카메라로부터 상대적으로 얼마나 떨어져 있는지 고려되어야 한다.<br/>이때 원근감은 고려되지 않는다.<br/><br/>직교투영 이라고도 한다 |
| 원근 투영 | 원근감을 살리기 위해 원근 투영한다.<br/>카메라가 직교투영모드인 경우 생략                                                           |

### 2.3 래스터라이즈
- 물체를 화면에 출력할 픽셀과 대응시키는 과정
  - = 3D 공간을 2D 화면으로 옮기는 과정
- 이를 위한 장치인 래스터라이저(Rasterizer)가 있다.

### 2.4. 픽셀 쉐이더 / 프래그먼트 쉐이더
- 픽셀 쉐이더(Pixel Shader) 혹은 프래그먼트 쉐이더(Fragment Shader)가 연산해 최종 색을 결정
  - 각종 조명, 텍스처, 그림자, 특수효과 등을 연산

## 3. Unity의 랜더링 파이프라인
- 렌더 파이프라인은 저마다 기능 및 성능, 특성을 지닌다.
  - 빌트인 렌더 파이프라인(Built-in Render Pipeline)
  - 스크립터블 렌더 파이프라인(SRP: Scriptable Render Pipeline)
  - 한 렌더 파이프라인에서 다른 렌더 파이프라인으로 프로젝트 전환이 어려울 수 있다.
- Unity에서 제공하는 랜더 파이프라인을 이해해야 개발 초기에 올바른 결정을 내릴 수 있다.

| 렌더 파이프라인      | 설명                                                                                                               |
|---------------|------------------------------------------------------------------------------------------------------------------|
| 빌트인 렌더 파이프라인  | Unity 기본 렌더 파이프라인<br/>범용으로 사용되나 커스터마이즈 옵션 제한적<br/>범용으로 사용하기 위해 다양한 플랫폼을 가정, 무겁다                                                                    |
| 유니버셜 렌더 파이프라인 | 광범위 플랫폼 최적화된 그래픽스 구현 지원 SRP<br/>이전에는 경량 렌더 파이프라인(LWRP: Lightweight Render Pipeline) 으로 불렸다.<br/>URP는 LWRP를 대체한다. |
| 고해상도 렌더 파이프라인 | 고사양 플랫폼 최신 고해상도 그래픽스 구현 지원 SRP                                                                                   |
| 커스텀 렌더 파이프라인  | 필요시 랜더 파이프라인 커스터마이징 가능                                                                                           |

## 4. PBS(= PBR)(Physically Based Shader/Rendering)
- 실제 물리학에 기반한 렌더링 기법
  - 정확하고 직관적인 라이팅 표현
  - 자동화가 많이 되고 계산이 좀 더 정밀해진 우버 셰이더에 가깝다.
  - 쉐이더 하나로 모든 재질 표현
  - 기존 라이트 방식은 정반사와 난반사를 따로 더해서 표현
- Specular Color
  - 금속은 고유의 스페큘러 컬러를 가진다.
  - 비금속은 그냥 흰색의 스페큘러 컬러를 가지고 있다.
  - PBS에서는 금속/비금속을 구분하는 기능이 생김
- 자동으로 출력되는 Albedo위치가 정해지는것이 PBS
- Unity는 전부 지원
  - 구형 라이트(non-PBS) : diffuse Ambient Specular
  - 신형 라이트(PBS) : BRDF Diffuse Metallic Smoothness
  - 신형 라이트 변형(PBS) : BRDF Diffuse Specular Color Smoothness
- 유니티의 빌트인 렌더 파이프라인은 PBR이다.
