# Node.js
## Node.js의 핵심 개념
- Node.js는 Chrome V8 Javascript엔진으로 빌드된 Javascript 런타임
  
### 서버
- 네트워크를 통해 클라이언트에 정보나 서비스를 제공하는 컴퓨터 또는 프로그램
- **클라이언트** : 요청을 보내는 주체

### 자바스크립트 런타임
- **런타임** : 특정 언어로 만든 프로그램들을 실행할 수 있는 환경
- 노드는 자바스크립트 런타임 => 노드는 자바스크립트 프로그램을 컴퓨터에서 실행할 수 있다.
  
### 이벤트 기반
- 이벤트가 발생할 때 미리 지정해둔 작업을 수행하는 방식
- 이벤트 : 클릭, 네트워크 요청
- 이벤트 기반 시스템에서는 특정 이벤트가 발생할 때 무엇을 할지 미리 등록을 해두어야 합니다
  - 이벤트 리스너에 콜백 함수를 등록한다.
  - ex) 클릭이벤트 리스터에 경고창을 띄우는 콜백 함수를 등록해두면 클릭 이벤트가 발생할 때 마다 콜백 함수가 실행
- 노드도 이벤트 기반 방식으로 동작한다. 발생한 이벤트가 없거나 발생했던 이벤트를 다 처리하면 노드는 다음 이벤트가 발생할 때까지 대기합니다.
- **이벤트 루프** : 이벤트 발생 시 호출할 콜백 함수들을 관리하고, 호출된 콜백 함수의 실행 순서를 결정하는 역할을 담당
- **백그라운드** : setTimeout 같은 타이머나 이벤트 리스너들이 대기하는 곳
  - **setTimeout** : 특정 밀리초 이후에 코드를 실행하는 함수
- **태스크 큐** : 이벤트 발생 후, 백 그라운드에서는 태스크 큐로 타이머나 리벤트 리스터의 콜백 함수를 보낸다.
> 만약 호출 스택에 함수들이 너무 많이 들어있으면 3초가 지난 후에도 run 함수가 실행되지 않을 수 있다.
### 논 블로킹 I/O
- **논블로킹** : 이전 작업이 완료될 때까지 대기하지 않고 다음 작업을 수행
  - 블로킹 : 이전 작업이 끝나야만 다음 작업을 수행하는 것을 의미
  - 이 방식이 작업을 더 빨리 처리할 수 있지만 전제조건 필요
  - 작업들이 동시에 처리될 수 있는 작업이어야 한다.
  
```javascript
// 블로킹 방식 코드
function longRunningTask()
{
    console.log('End');
}
console.log('Start');
longRunngingTask();
console.log('Next');
// Start -> End -> Next
// 논블로킹 방식 코드

function longRunningTask()
{
    console.log('End');
}
console.log('Start');
setTimeout(longRunngingTask, 0);
console.log('Next');
// Start -> Next -> End
```

- 논 블로킹과 동시가 같은 의미가 아니다.
- 블로킹은 동기와 유사, 논 블로킹은 비동기와 유사
  
### 싱글 스레드
- 스레드가 하나뿐인 것을 의미
- 프로세스 : 운영체제가 할당하는 작업의 단위
- 스레드 : 프로세스 내에서 실행되는 흐름의 단위

#### 스레드 풀과 워커 스레드
- 노드가 싱글 스레드로 동작하지 않는 두 가지 경우
  - 스레드 풀 : 노드가 특정 동작을 수행할 때 스스로 멀티 스레드를 사용 (암호화, 파일 입출력, 압축 등)
  - 워커 스레드 : 노드 12 버전에서 안정화된 기능으로 노드에서도 멀티 스레드를 사용 가능
- 멀티 스레드가 싱글 스레드보다 무조건 좋다? -> 무조건 그렇진 않다.
  - 싱글 스레드, 블로킹 : 작업이 끝날때까지 다음 작업이 대기를 하는 상태 (매우 비효율적)
  - 싱글 스레드, 논 블로킹 : 작업이 들어왔다는 것은 계속 알려주고, 작업이 끝나는 순서대로 반환 (노드에서 사용하는 방식)
  - 멀티 스레드, 블로킹 : 작업이 늘어날 수록 사용량이 늘고, 중간중간 노는 스레드가 발생(유지비용이 크다)

## 서버로서의 노드
- 노드는 기본적으로 싱글 스레드, 논 블로킹 모델을 사용하므로 노드 서버 또한 동일한 모델
- 노드는 CPU 부하가 큰 작업에는 적합하지 않다.
  - 코드가 CPU연산을 많이 요구하면 싱글 스레드로서는 감당하기 어렵다.
- 실시간 채팅 애플리케이션이나 주식 차트. JSON 데이터를 제공하는 API서버가 노드를 많이 사용합니다.
- 노드 12 버전에서 노드로 멀티스레드 작업을 할 수 있게 되었지만, 프로그래밍적으로 어려워 진다.
- 노드에는 웹 서버가 내장 되어 있다.
  - 단, 서버의 규모가 커지면 웹 서버를 연결해야한다.
- 노드의 장점
  - 자바스크립트를 사용한다는 것
  - 하나의 언어로 웹 사이트를 개발 할 수 있다. -> 개발 생산성을 높혔다.
- 노드의 단점
  - 다른 웹 서버에 비해서 속도가 느리다.
