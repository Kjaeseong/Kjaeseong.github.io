# npm(Node Package Manager)
- 미리 작성된 공개 코드들이 업데이트 되는 서버
  - 여기 업로드된 노드 모듈을 패키지라 한다
  - 패키지가 다른 패키지를 사용할 수도 있으며, 이런 관계를 의존 관게라 한다.
- 특정 기능 패키지가 필요하다면 npm에서 찾아 설치하면 된다.
- npm의 대체자로 페이스북이 내놓은 yarn도 있다.

## package.json
- 패키지의 버전을 관리하는 파일
- 패키지는 저마다 고유한 버전이 있으므로 어딘가에 기록해야 하낟.
  - 같은 패키지라도 버전별로 기능이 다를 수 있다
  - 설치할 때 패키지도 동일버전을 설치하지 않으면 문제발생 여지 있음
- 노드 프로젝트를 시작하기 전 폴더 내부에 무조건 package.json부터 만든다.
- console에 `npm init` 입력시 설치 진행
  - 몇 가지 명령어와 함께 프로젝트 정보를 기입하게 된다.

| 명령어            | 설명                                                                                        |
|----------------|-------------------------------------------------------------------------------------------|
| package name   | 패키지 이름<br/>package.json의 name 속성에 저장된다                                                    |
| version        | 패키지 버전<br/>npm버전은 다소 엄격하게 관리된다                                                            |
| entry point    | 자바스크립트 실행 파일 진입점<br/>보통 마지막으로 module.exports하는 파일 지정<br/>package.json의 main속성에 저장         |
| test command   | 코드 테스트시 입력할 명령어<br/>package.json의 scripts 속성 안의 test 속성에 저장                               |
| git repository | Git 저장소 주소 의미<br/>package.json의 keywords 속성에 저장                                           |
| keywords       | [npm 공식 홈페이지](https://npmjs.com)에서 패키지를 쉽게 찾을 수 있도록 한다.<br/>package.json의 keywords 속성에 저장 |
| license        | 해당 패키지의 라이센스 기입                                                                           |

### 라이센스
- 라이선스별로 특징이 다르므로 오픈 소스를 사용하기 전 반드시 세부내용까지 확인해야 한다.
  - 상용 프로그램 개발 시 법적 문제가 생길 수 있다.

| 라이센스 | 내용                                         |
|------|--------------------------------------------|
| ISC  | 사용한 패키지, 라이센스를 밝히면 자유롭게 사용 가능              |
| MIT  | 사용한 패키지, 라이센스를 밝히면 자유롭게 사용 가능              |
| BSD  | 사용한 패키지, 라이센스를 밝히면 자유롭게 사용 가능              |
| 아파치  | 사용은 자유롭지만 특허권에 대한 제한 포함                    |
| GPL  | (주의)배포시 자신의 패키지도 GPL로 배포하고 소스 코드도 공개해야 한다. |
